Formo is a form module for Kohana 3 that lets you work with forms as objects.

Installation

Since Formo was written specifically for Kohana, installation is a breeze. To install Formo in your application: 1. Make sure you are running PHP 5.3.x or greater 2. Create a directory for Formo in your modules directory 3. Clone Formo (git://github.com/bmidget/kohana-formo.git) or copy Formo into your new directory 4. Add the Formo module to the modules located in /application/bootstrap.php That’s it, you’re ready to use Formo.

Go to the Github Repo

## Extending Formo Core Like all things Kohana, Formo is made to be transparently extended. Extend the crap out of any class the same way you do any Kohana classes. Note that Formo follows a different standard for it's filesystem. Rather than placing its core files in `classes/kohana/{file}`, Formo's core files are located in `classes/formo/core/{file}`.

Upgrading

Here are some changes that you need to know in order to upgrade from the last version of Formo - Rules are no longer added using `Formo::rule()` or `Formo::rules()`. Instead you must use `Formo::add_rule()`. [See validation documentation](validation.html) - There is no longer a `Formo::field()` method. Just use `Formo::form()` and `Formo::factory()` - `create_sub()` is now `subform()`, and it returns the form object, not the subform - Formo doesn't do automatic belongs to, has many, or has many through relationships because it's ridiculous to auto-generate these - File fields have a `NULL` empty value, not an empty `$_FILES` array - Out-of-the box styling works now with Twitter Bootstrap. If you need backwards compatibility with the old version of Formo, ste the `template_dir` variable in Formo's config file to `'formo/'` - Views are not located in `views/formo/html/{file}`, they are instead located in `template_dir/template` - The array of values for select options, radios and checkboxes is no longer called `options` and is now called `opts`. - If creating a field using an array of key/value pairs, use `val` instead of `value` as the option for the field's value - `bool` driver is now `checkbox` - `hidden`, `password`, `submit` are now `input|hidden`, `input|password`, `input|submit` - `Field::set_all($array)` is now `Formo::set_fields($array)` - `Formo_Form::add_group()` has been deprecated. Now just use `Formo::add()` and set the `opts` array for option values - Formo no longer auto-loads *belongs to*, *has many*, *has one* or *belongs to* relationships - Button fields no longer automatically add any html inside the button tag. To do it, set the field's HTML. (See HTML helper methods)
